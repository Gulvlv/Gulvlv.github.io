Number Theory and Cryptographic Hardness Assumptions数论和密码学困难假设

现代密码学系统总是基于一个困难问题提出的，例如：RSA基于大整数因子分解问题、Diffie-Hellman基于有限域上的‌离散对数问题‌、ECC基于椭圆曲线上的离散对数问题等等…

引出One-way Function单向函数，以及基于何种大数困难问题。 → 是否就是引出One-way Trapdoor Function单项陷门函数？

基础数论知识预备

后续是一些简单的数论问题在密码学中的运用

素数和可除性

  1. a|b → a 整除 b → b=ka
    同时若 a∉{1,b} 那么 a 称为 b 的一个非平凡除数 → 1和它本身称为平凡约数；大于1小于它本身的约数称为非平凡约数。整数b的非平凡约数(nontrivial divisor)称为b的因数(factor)。
    a 是一个整数，设 b 是一个正整数，那么存在唯一的整数 q、r ，使得 a=qb+r 并且 0≤r<b 。

  2. 最大公约数 c 记作 gcd(a,b) ，使得 c|a 和 c|b 。如果 gcd(a,b)=1 ，a和b是互质的。
    设 a,b 为正整数，则存在 X,Y 使得 Xa+Yb=gcd(a,b) 。此外， gcd(a,b) 是能以这种方式表示的最小正整数。

  3. c|ab 和 gcd(a,c)=1 ，那么 c|b 。因此，如果 p 是素数且 p|ab ，则要么 p|a or p|b 。

  4. 如果 a|N ， b|N ，和 gcd(a,b)=1 ，那么 ab|N 。

模运算

同余关系： a 和 b 模 N 同余，记作 a=bmodN 即，即 [amodN]=[bmodN]
同余有自反性，传递性,同余也遵守关于加法、减法和乘法的标准算术规则。注：无除法， ab=cbmodN 不一定意味着 a=cmodN 。

素数、分解和RSA

正如前面所说，这里引出RSA，RSA是基于大整数分解困难问题提出的公钥。原文引出整数分离困难，再紧接着提出最难分解的数是那些只有大素数因子的数。

生成随机素数

找n位中的素数，用循环一次一次找n位随机数，对比是否为素数。
ALGORITHM RANDOM算法伪代码：

引申出素数定理，数学中的一个重要结果，给出了给定长度的整数中素数比例的相当精确的界限。
对于任何 n>1 ， n 位整数为素数的比例至少为 1/3n 。

设置t=3n2，那么在所有t次算法迭代中，一个素数没有被选中的概率最多是

(1−13n)t=((1−13n)3n)n≤(e−1)n=e−n

在n中是可忽略的。因此，使用poly(n)次迭代，得到一个算法，其输出失败的概率在n中是可忽略的。

素性测试

如果 p 是素数，那么Miller–Rabin测试总是输出素数。如果 p 是合数，算法除以概率至多为 2−t 外，总是输出合数。

引出

  1. 设 G 是一个有限群，且 H⊆G 。假设H非空，并且对于所有的 a , b∈H 都有 ab∈H 。则 H 是 G 的一个子群

  2. 设 H 是有限群 G 的一个严格子群（即 H≠G ）。则 |H|≤|G|/2

  3. 假设存在一个witness（见证者）使得 N 是合数。那么 ZN∗ 中至少有一半的元素是见证者使得 N 是合数。

  4. 设 x∈ZN∗ 是 1modN 当 x2 =1modN 的平方根。如果 N 是奇素数（不是 2 的素数），那么 1 mod N 的唯一平方根是[ ±1 mod N ]。

  5. 设 N 是一个不是素数幂的奇数, 那么 ZN∗ 中至少有一半的元素是强见证，证明 N 是合数。

补充：

  1. prime power（素数幂）: N=pk （其中 p 是素数， k≥1 且 k 为整数）。

  2. 对于理解 5. 设 N 是一个奇数，并且它不能写成 pk 的形式（其中 p 是素数， k≥1 且 k 为整数）。那么 ZN∗ 中至少有一半的元素是强见证，证明 N 是合数。

分解假设

N=p∗q
分解实验FactorA,GenModulus(n)：

  1. 运行 GenModulus ( 1n )以获得 (N,p,q) 。
  2. A被给定N，并输出 p′,q′>1 。
  3. 当 p′∗q′=N ,实验的输出定义为1，否则为0。

RSA假设

分解假设确实给出了一种单向函数,但它并不能直接产生实用密码系统。促使人们寻找其他难度与分解难度相关的问题，最著名的是由Rivest、Shamir和Adleman于1978年引入的RSA问题。
设GenRSA是一个概率多项式时间算法

获得 (N,e,d) 当 xe=ymodN 时输出为1，否则为0。

RSA问题相对于GenRSA是困难的。RSA假设是存在一个GenRSA算法，相对于该算法，RSA问题是困难的。一个合适的GenRSA算法可以从任何生成合数模数及其因数分解的算法GenModulus中构造出来。
引申公式

  1. φ(n)=(p−1)∗(q−1)
  2. d∗e≡1modN
  3. N=p∗q
    即当 d 或 N 的因数分解未知时RSA问题似乎很困难，但当 d 已知时变得容易——构成了RSA问题应用于公钥密码学的基础。

补充

  1. 关于 e 的选择， e=216+1=65537

  2. 基于RSA问题的密码系统不良实现所产生的结果，选择d过于小，因为当d位于一个非常小的范围内，那么可以对d进行暴力搜索。

将分解与中国剩余定理与RSA假设联系起来

d=[e−1modφ(N)] , e 是 d 的逆元。
所以，为了使RSA问题相对于GenRSA是困难的，分解问题必须相对于GenModulus是困难的。换句话说，RSA问题不能比分解更难；分解的难度（相对于GenModulus）只能潜在地比RSA问题的难度（相对于GenRSA）更弱。

推论： 存在一个概率多项式时间算法，给定作为输入一个整数 N 和整数 e,d ，其中 ed=1modφ(N) ，除了在 ||N|| 中概率可忽略地外，分解 N 。
